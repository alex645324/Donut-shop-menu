KEY IMPLEMENTATION DETAILS
================================================================================

✅ Transaction ID Generation:
   Format: OD- + 8-character UUID (e.g., OD-A1B2C3D4)
   Unique constraint enforced in database

✅ Date/Time Tracking:
   Automatic timestamp on order creation
   Stored as TIMESTAMP in Derby
   Displayed as: yyyy-MM-dd HH:mm:ss

✅ Order Items Storage:
   Items linked via ORDER_ITEMS junction table
   Foreign keys ensure referential integrity
   Cascading delete on order removal

✅ MVVM Architecture:
   Models: MenuItem, Order (pure data)
   ViewModel: OrderController (business logic)
   Views: GUI panels (UI only)
   Database: DatabaseManager (data access)
